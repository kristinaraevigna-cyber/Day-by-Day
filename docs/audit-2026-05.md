## TL;DR

The real app is a working leader-development app in **a single 7,820-line `src/App.jsx`** with:
- 8 fully-written chapters from Day's book (industry-neutral content)
- 8 implemented assessment instruments (full items + scoring, no charts)
- Two AI coaches: ICF (voice via OpenAI Realtime + text via Supabase edge function) and Day Advisor (text via Supabase edge function; voice path shares the OpenAI Realtime component)
- Journal, Actions, Reading Library, Practice activities (VCoL-based), Consent gate, Privacy settings
- Supabase auth + Postgres (no schema files in repo)
- State-driven view switching (`currentView`), no React Router
- Zero tests, zero TODOs in code, no `.env.example`

**Three critical corrections to the prompt's premise:**

1. **There is no `currentWeek` hardcoded-to-1 bug.** Grep returns zero matches. The "phase 1/2/3" logic that exists is *per-intervention* (each developmental intervention has 3 phases like "Identity Awareness → Identity Claiming → Identity Integration"), not a global pre/mid/post system. The prompt's described bug doesn't exist because the surrounding global-phase system doesn't exist at all.

2. **The app is industry-neutral, not first-responder-coded.** A search for first-responder / police / fire / military / executive / CEO / CFO returned only one hit: the word "executives" in a single book description in the reading library. No content needs to be *replaced* — it needs to be *contextualized* for healthcare.

3. **The Areas of Worklife Survey (AWS) is not implemented.** The prompt names AWS as the demo centerpiece — it isn't in the code. Eight other instruments are fully implemented. Adding AWS is net-new work, not "wire up the radar chart on existing scoring."

The Phase 0 paths in the prompt need re-scoping against these realities. See `plan-2026-05.md`.

### 4. The `currentWeek` "bug"

`grep -n currentWeek src/App.jsx` → **zero matches.**

`grep -niE 'baseline|midpoint|post-?assessment'` → matches only inside per-intervention copy (e.g., "baseline awareness check" as the first phase of the Leader Identity intervention), not as global pre/mid/post system constants.

**Conclusion:** The bug as described in the prompt does not exist because the surrounding feature does not exist. Building pre/mid/post phasing is Phase 2 net-new work, not a bug fix.

### 5. AI coaches

**ICF Coach.**
- Voice: OpenAI Realtime API (`gpt-4o-realtime-preview-2024-12-17`) via WebRTC. Session token minted by `voice-session` edge function (line 3449). System prompt at lines 13–49.
- Text: Calls `coach` edge function (line 3715). Edge function is not in repo; it's deployed in Supabase.
- Session save: `coaching_sessions` table, columns `user_id, coach_type='icf', session_type, started_at, ended_at, duration_seconds, messages, include_in_research` (line 3317).
- Action extraction: Looks for ```actions [JSON]``` fenced block at end of AI response; falls back to regex (lines 3343–3373). Writes to `actions` table.

**Day Advisor.**
- Voice: same `VoiceCoach` component, `coachType="mentor"` (so OpenAI Realtime, primed with Day's research). Voice prompt at lines 51–89.
- Text: `day-advisor` edge function (line 3715) — Anthropic Claude.
- Same session/action storage pattern.

Both coach prompts are deliberate about voice constraints (2–4 sentences, one question per turn), ICF non-directive vs Day directive/research-based, and structured action capture.

### 6. Journal, Actions, Library, Practice

- **Journal** (lines 4045–4158): Freeform textarea + 3 standing reflection prompts; rows in `reflections` (`user_id, content, reflection_type, created_at`). No editing of saved entries.
- **Actions** (lines 4160–4250): Commitments captured from coach sessions or entered manually. Toggle complete / delete. Table `actions` (`user_id, action, timeline, source, completed, completed_at`).
- **Library** (lines 704–839, 4252–): 9 categories, ~50 books with title/author/year/description/keyTopics/evidenceLevel/recommended/primary flags. Day's 2024 book is `primary: true`.
- **Practice activities** (lines 851–1552): 45+ short activities aligned to the VCoL (verify-calibrate-learn) framework. Tagged by competency (Courage / Creativity / Collaboration).

### 7. Onboarding / first-login

Flow on auth:
1. Supabase auth via `AuthScreen` (lines 3199–3266) — email/password.
2. `useEffect` on user (line 7725) calls `checkConsent()`.
3. If `hasConsent === false`, render `ConsentScreen` (lines 7758–7766). User must accept privacy policy + data-processing consent. Optional research-participation opt-in.
4. After consent, land on `Dashboard`.

**No role selection. No baseline-assessment gate. No enrollment timestamp captured.** This is the onboarding redesign target for Phase 4.

### 9. Healthcare reframing surface

Search of `App.jsx` for first-responder / police / fire / military / soldier / executive / CEO / CFO / VP returned **one hit**: the word "executives" in one library book description (line 732, *The Lessons of Experience*).

The content is industry-neutral. "Healthcare reframing" therefore means **adding** healthcare context (scenarios, examples, vocabulary, coach guardrails, role selection during onboarding) on top of the current abstract framing — not replacing wrong framing. That's a smaller content-rewrite job than the prompt implies, but it's still real work across:
- Chapter copy (8 chapters × ~5 lessons each)
- Practice activity examples (45 activities)
- Reflection prompts (journal + per-chapter)
- Coach system prompts (add healthcare terminology + guardrails)
- New: role selection in onboarding
